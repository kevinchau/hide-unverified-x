Chrome Web Store listing copy for Hide Unverified X
====================================================

Short description (132 character max)
-------------------------------------
Filter unverified X posts by badge & About-account region. Hide, dim, whitelist follows. Local only, no API keys.

Store summary / promo text (250 characters)
--------------------------------------------
Filter unverified X posts by blue, gold, or silver badge and About-account region (e.g. App Store). Hide or dim on For You, Following, and Replies. Whitelist follows and friends-of-follows. Blocklist spam regions. Local only—no API keys or tracking.

(249 characters)


Permission justifications (for reviewer form)
---------------------------------------------
storage     — Save filter settings and whitelist (via storage.sync where available), plus local caches: About-account lookups, following handles, and followed-by-following handles used for whitelist features.
activeTab   — Read the filtered-post count for the popup on the tab where the user opens the extension (no broad tabs access).

Host permissions: none declared. Content scripts run only on x.com and twitter.com.


Single purpose
--------------
Filter posts on X/Twitter by verification badge and About this account metadata to reduce spam in the user's feed.


Privacy
-------
No data collected, transmitted, or sold to the extension author. No third-party analytics.

Local device storage only:
- Settings and whitelist
- About-account lookup cache
- Following handles cache
- Followed-by-following handles cache

Settings may sync across the user's signed-in browser profile via storage.sync (browser-managed, not sent to the developer).

When the About-account filter is enabled, the extension may call X GraphQL using the user's existing logged-in X session to resolve account origin; results stay in the local cache. No API keys.

Firefox builds check for updates via GitHub Releases / updates.json. Chrome Web Store builds do not use that self-update path.
